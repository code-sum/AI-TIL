### [CVPR 2025] Flowing from Words to Pixels: A Noise-Free Framework for Cross-Modality Evolution [🗂️](https://arxiv.org/abs/2412.15213)


- Diffusion & Flow Matching
  - Diffusion
    - Gaussian noise > target data 분포로의 매핑 학습
    - Cross-modal (예: Text-to-Image) 에서는 텍스트가 조건(condition)으로 추가됨 (주로 cross-attention 활용)
  - Flow Matching
    - Diffusion 의 일반화 버전 (더 넓은 개념)
    - Noise 에서 target 으로 가는 더 단순하고 최적화된 경로 학습
    - Source 분포가 반드시 noise 일 필요는 없음 > target 과 상관관계 있는 다른 분포 (예: text latent) 사용 가능

- Background
  - RQ: Flow Matching 방식을 사용해 Noise, Condition 없이 직접 modality 간 매핑이 가능할까?
  - (1) Shape Mismatch
    - Diffusion/Flow 모델은 source 와 target 분포의 shape 이 동일해야 함
    - ➡️ 서로 다른 모달리티(예: 텍스트 vs 이미지)는 구조가 달라서 직접 매핑 불가
  - (2) Classifier-Free Guidance(CFG) 적용 불가
    - 기존 SOTA 모델들은 CFG 로 성능 향상
    - Flow Matching 방식 채택 시, 조건 정보가 source data 자체에 포함되어 있음
    - ➡️ 기존 CFG 방식(조건 on/off 전환) 사용 불가능

- Methods: Variational Encoder
  - 문제점
    - Flow Matching 은 source 분포와 target 분포가 같은 shape 이어야 함
    - 단순 Encoder 로 직접 벡터 인코딩 시 입력 정보를 보존할 수는 있지만 flow matching 프레임워크에서는 불안정
    - ➡️ 정규화된 latent 분포가 필요
  - 해결책: Variational Encoder (VE)
    ```math
    z_0 \sim \mathcal{N}(\bar{\mu}_{z_0}, \bar{\sigma}_{z_0}^2)
    ```
    - 단일 벡터를 입력하는 것이 아니라, 분포의 평균과 분산 예측 후 정규화된 분포에서 샘플
  - 학습방식
    - Loss = Encoding Loss + KL Divergence Loss
    - 일반 VAE 와 달리 Decoder 는 필요 없음 (Encoder 역할만 수행)

- Methods: Training CrossFlow
    ```math
    \begin{aligned}
    L &= L_{FM} + L_{Enc} + \lambda L_{KL} \\
      &= \mathrm{MSE}(v_\theta(z_t, t), \hat{v}) + \mathrm{Enc}(z_0, \hat{z}) \\
      &\quad + \lambda \mathrm{KL}(\mathcal{N}(\bar{\mu}_{z_0}, \bar{\sigma}_{z_0}^2) \parallel \mathcal{N}(0, 1))
    \end{aligned}
    ```
  - Flow Matching MSE Loss: 올바른 velocity 예측
  - Encoding loss: task 에 따라 상이
  - KL Divergence: latent 분포를 N(0,1)에 가깝게 정규화

- Arithmetic Operations in Latent Space
  - Latent Interpoltation
    - 텍스트 A와 텍스트 B를 인코딩
      ```math
        Z_{A}, Z_{B}
      ```
    - 두 latent를 linear interpolate
      ```math
        \lambda Z_{A} + (1-\lambda) Z_{B}
      ```
  - Latent Arithmetic Operation
    - 단순 덧셈/뺄셈으로 의미적 속성 제어 가능
    - Latent space에서 연산이 출력 이미지의 의미 변화를 직접 반영
  - CrossFlow 의 Text VE 가 semantic 구조를 잘 보존하면서도 통일된 latent 표현을 만듦
    - 따라서 latent 공간에서의 연산이 단순한 수치적 조작이 아니라, 의미적인 조작으로 이어짐
