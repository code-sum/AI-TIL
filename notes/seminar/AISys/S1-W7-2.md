### [MLSys 2025] ThunderServe: High-performance and Cost-efficient LLM Serving in Cloud Environments [🗂️](https://arxiv.org/abs/2502.09334)

- 3줄 요약
  - ThunderServe는 이기종 GPU와 다양한 네트워크 환경을 가진 클라우드에서 LLM을 고성능으로 비용 효율적으로 서빙하기 위한 시스템을 제안.
  - 이 시스템은 이기종 GPU 자원 및 네트워크 대역폭을 최적화하는 새로운 스케줄링 알고리즘과 워크로드 변화에 유연하게 대응하는 경량 재스케줄링 메커니즘을 포함하며, KV cache 압축 기술로 통신 오버헤드를 줄임.
  - 실험 결과, ThunderServe는 동일한 가격 예산으로 기존 시스템 대비 최대 2.1배의 Throughput 증가와 최대 2.5배의 Latency deadline 감소를 달성하여 클라우드 LLM 서빙의 비용 효율성을 입증.

- Background
  - 최근 LLM 서비스들이 시장에 쏟아지고 있지만, LLM 을 제대로 서빙하려면 고가의 GPU 를 대량으로 갖춰야 함.
  - 엔비디아 A100 같은 고성능 GPU 는 공급이 부족하고 비싸기 때문에 service provider 입장에선
    - 정해진 예산 안에서 동일한 GPU 를 어떻게든 확보해서 인하우스 클러스터를 구축할지 (homogeneous in-house)
    - 아니면 다양한 GPU 를 조합해야 하는 risk 를 무릅쓰고 클라우드 환경에 LLM 서빙 시스템을 갖출 것인지 (heterogeneous cloud) 선택해야 함.
  - 클라우드 활용에 대한 고민을 조금이라도 해소하기 위해 저자들은 ThunderServe 라는 스케줄링 알고리즘을 제안.
